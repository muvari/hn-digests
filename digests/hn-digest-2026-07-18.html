1 Security

LG monitors silently install software through Windows Update without consent

1,173 points · 586 comments · videocardz.com

Gamers Nexus found that connecting certain LG monitors to a Windows PC triggers Windows Update to silently install an "LG Monitor App Installer" with no consent prompt, which then displayed a McAfee subscription trial popup on 31 of 32 test boots. The behavior wasn't limited to new hardware — it also appeared on a three-year-old UltraFine monitor, and user reports of it date back to 2024. Dell does something similar, pushing Alienware Command Center through Windows Update when it detects compatible hardware. Users can block the behavior via a Windows Group Policy or Device Installation Settings change, though this may also block legitimate driver software.

What the comments said

  • Several commenters confirmed experiencing this themselves, with the installer and McAfee promo only visible after the fact via the Microsoft Store's Library.
  • Many noted LG isn't unique here — Razer, Logitech, Nvidia and other peripheral makers push similar bundled software through Windows Update, and getting just the driver without the userspace app is often difficult or impossible.
  • Commenters shared concrete workarounds: the gpedit.msc "Prevent automatic download of applications associated with device metadata" policy, or the Device Installation Settings toggle on Home editions that lack gpedit.
  • Strong sentiment that this fits the textbook definition of malware/spyware, with debate over whether the decision originated from an engineering team, a marketing/partnership deal, or was pushed down from leadership.
  • Some argued Microsoft itself needs to intervene, since the workaround isn't available by default on Home editions and shouldn't be a normal expectation for using the OS.

4 Programming

Regressive JPEGs

696 points · 68 comments · maurycyz.com

The author dissects JPEG's progressive-scan feature, where an image is split into multiple scans covering different frequency bands so a partial download renders as a blurry preview rather than a cutoff image. By concatenating the scan data of several same-resolution images (stripping conflicting markers), they show a single JPEG file can be made to switch between different pictures as it streams over a slow connection, since later scans overwrite earlier decoded data. Pushing further, they build JPEGs out of many single-scan "DC-only" frames — each independently a valid, standards-compliant low-res image — via jpegtran, letting them pack dozens of frames (~90 in Chrome, more in Firefox) into one file to fake crude streamed video, including a rickroll and a Bad Apple-style playback driven purely by partial JPEG rendering.

What the comments said

  • Commenters were delighted, calling the technique "cursed" but exactly the kind of hack they come to HN for.
  • Several noted prior art for similar streaming-image tricks, like Motion JPEG and the multipart/x-mixed-replace MIME type used by IP cameras and webcams, which achieve comparable "replace the image" behavior without any JavaScript.
  • One suggested extending the idea to live video by generating the JPEG on the fly server-side (even from a webcam) and pacing chunks to control timing, since the current technique has no built-in timing information.
  • A few discussed doing an analogous trick with animated GIFs, including a past experiment combining a frame-at-a-time GIF with an image map to build a JavaScript-free "remote desktop."
  • A wry aside speculated that big tech would find a way to repurpose the trick for browser or device fingerprinting.

  • Several generalized to distrust of cheap, Chinese-made IoT hardware, though others countered that insecure IoT coding isn't unique to Chinese vendors.

Vāgdhenu: A Sanskrit Chanting TTS System

212 points · 53 comments · prathosh.in
